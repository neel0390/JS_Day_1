4.What is execution context


Every Browser has a JS engine.For Example,Google has V8 engine,Internet Explorer has chakra engine and Mozilla firefox has Spider Monkey engine.etc... Js engine uses call stack and also creates a special environment to handle the execution of javascript code.This Environment is known as Execution Context.It contains the code that is curently running and everthing aids in execution.

During Execution specific code is parsed by the parser and the variables and functions are stored in memory,executeable byte code generated and the code gets executed.

In JS, we have two kinds of Execution context.
1.Global Execution context(GEC)

2.Function Execution context.(FEC)

The moment code gets executed GEC gets created and pushed into call stack.Inside GEC and FEC we have 2 phases namely.

1.Memory Allocation(creation phase)

2.Code execution(Execution phase)

In creation phase the memory gets allocated for variables and functions.

  • variables with var keyword gets memory allocated as undefined as it is global scoped.
  • variables with let and const keyword gets memory allocated as undefined as they are block scoped.
  • If we try to access the var variables before intialization then we get undefined.If we access let or const variables before intialization then we will get Reference error bcz they are block scoped and cannot be accesed without declaration.
  • In Execution phase functions get memory allocated as their actual value defined in code and in code execution variables get their actual value assigned to them and function will create FEC.the moment FEC gets created it will be pushed into call stack and once execution completed it gets removed from call stack and point of execution moved to GEC.
